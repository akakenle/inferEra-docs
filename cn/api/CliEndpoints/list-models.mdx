---
title: "获取所有可用模型列表"
description: "返回所有可用模型（可能按渠道/分组返回）"
api: "GET /api/models"
---

## 请求参数

### 请求头（headers）

<ResponseField type="string" required>
  请求身份认证。接口使用 AIHubMix Manage Key 进行身份认证。
</ResponseField>

<ResponseField type="string">
  请求内容类型。此参数必须设置为 application/json。
</ResponseField>

## 响应参数

<ResponseField name="success" type="boolean">
  是否成功。
</ResponseField>

<ResponseField name="message" type="string">
  提示信息（可能为空）。请求失败时返回失败原因。
</ResponseField>

<ResponseField name="data" type="object">
  模型集合。通常为“分组/渠道”到“模型数组”的映射结构（字段以实际返回为准），例如： `channel_id -> model[]` 或 `group -> model[]`。

  <ResponseField name="* (channel_id / group)" type="array">
    某个分组下的模型列表（数组中每个元素为一个模型对象）。

    <ResponseField name="id" type="string">
      模型标识（如：deepinfra-gemma-3-12b-it）。
    </ResponseField>
    <ResponseField name="name" type="string">
      模型展示名称（如接口有该字段）。
    </ResponseField>
    <ResponseField name="channel" type="string">
      所属渠道/来源（如接口有该字段）。
    </ResponseField>
    <ResponseField name="type" type="string">
      模型类型（如 chat / embedding / image 等，若接口有该字段）。
    </ResponseField>
    <ResponseField name="description" type="string">
      模型描述（如接口有该字段）。
    </ResponseField>
  </ResponseField>
</ResponseField>
