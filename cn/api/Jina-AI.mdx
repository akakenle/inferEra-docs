## 说明
我们集成了 Jina AI 的三个核心接口，助你轻松构建功能强大的智能体。这些接口主要适用于以下场景：
- **向量嵌入 (Embeddings)**：适用于多模态 RAG 问答场景，例如智能客服、智能招聘和知识库问答。
- **重排序 (Rerank)**：通过优化 Embedding 候选结果，依据话题相关性进行重排序，显著提升大型语言模型的回答质量。
- **深度搜索 (DeepSearch)**：进行深度搜索与推理，直至找到最优答案，特别适用于课题研究和产品解决方案制定等复杂任务。

## 快速指引
除了更换 `API_KEY` 为 [AiHubMIX_API_KEY](https://aihubmix.com/token) 和模型端点链接，其他参数和用法和 [Jina AI 官方](https://jina.ai/)完全一致。

**端点替换：**
- **向量嵌入 (Embeddings)**：`https://jina.ai/embeddings` -> `https://aihubmix.com/v1/embeddings`
- **重排序 (Rerank)**：`https://api.jina.ai/v1/rerank` -> `https://aihubmix.com/v1/rerank`
- **深度搜索 (DeepSearch)**：`https://deepsearch.jina.ai/v1/chat/completions` -> `https://aihubmix.com/v1/chat/completions`

## 一、向量嵌入 (Embeddings)
Jina AI 的 Embedding 支持普通文本和多模态图文，对于多语言任务的处理表现出众。

### 请求参数
<ParamField body="model" type="string" required>
  模型名称，可用模型列表如下：
  - `jina-clip-v2`：多模态、多语言、1024 维、8K 上下文窗口、865M 参数
  - `jina-embeddings-v3`：文本模型、多语言、1024 维、8K 上下文窗口、570M 参数
  - `jina-colbert-v2`：多语言 ColBERT 模型，8K token 上下文，560M 参数，用于嵌入和重排序
  - `jina-clip-v1`：多模态模型，支持图像和英文文本，768 维，8K 上下文窗口，223M 参数
  - `jina-colbert-v1-en`：英文 ColBERT 模型，8K token 上下文，137M 参数，用于嵌入和重排序
  - `jina-embeddings-v2-base-es`：西班牙语 - 英语双语嵌入模型，768 维，8K 上下文窗口，161M 参数
  - `jina-embeddings-v2-base-code`：针对代码和文档搜索优化的模型，768 维，8K 上下文窗口，137M 参数
  - `jina-embeddings-v2-base-de`：德语 - 英语双语嵌入模型，768 维，8K 上下文窗口，161M 参数
  - `jina-embeddings-v2-base-zh`：中文 - 英语双语嵌入模型，768 维，8K 上下文窗口，161M 参数
  - `jina-embeddings-v2-base-en`：英文嵌入模型，与 OpenAI 的 text-embedding-ada002 性能相当，768 维，8K 上下文窗口，137M 参数
</ParamField>

<ParamField body="input" type="array" required>
  输入文本或图片，根据不同模型支持不同的输入格式。对于文本，直接提供字符串数组；对于多模态模型，可以提供包含 text 或 image 字段的对象数组
</ParamField>

<ParamField body="normalized" type="boolean" default="true">
  是否进行 L2 归一化。L2 归一化会将嵌入向量的欧几里得范数缩放为 1，同时保留方向。当下游任务涉及点积、分类或可视化时很有用
</ParamField>

<ParamField body="embedding_type" type="string" default="float">
  返回的数据类型，可选值：
  - `float`：默认，返回浮点数数组。最常见且易于使用的格式，返回为浮点数列表
  - `binary_int8`：返回为 int8 打包的二进制格式。更高效的存储、搜索和传输方式
  - `binary_uint8`：返回为 uint8 打包的二进制格式。更高效的存储、搜索和传输方式
  - `base64`：返回 base64 编码的字符串。更高效的传输方式
</ParamField>


<ParamField body="embed_as" type="string" default="query">
  指定输入文本在搜索中的角色，可选：
  - `query`：查询嵌入，用于搜索查询
  - `document`：文档嵌入，用于被索引的内容
  
  同一段输入文本可作为查询或文档嵌入，取决于其在搜索中的角色
</ParamField>

<ParamField body="compatible_mode" type="boolean" default="false">
  是否使用与纯文本嵌入模型兼容的请求格式。这允许您在不更改请求格式的情况下在不同模型之间切换。
  注意：此模式下不支持图像输入
</ParamField>

## 二、重排序 (Rerank)
重排序器的目标是提高搜索相关性和 RAG 准确性。它通过对初始搜索结果的深度分析，考虑查询与文档内容之间的细微交互，从而重新排列搜索结果，将最相关的结果放在顶部。

### 请求参数
<ParamField body="model" type="string" required>
  模型名称，可用模型列表如下：
  - `jina-reranker-v2-base-multilingual`：最新的多语言重排序器，278M 参数，8K 上下文
  - `jina-reranker-m0`：多模态多语言文档重排序器，10K 上下文，2.4B 参数，用于视觉文档排序
  - `jina-colbert-v2`：多语言 ColBERT 模型，8K 上下文，560M 参数，用于嵌入和重排序
  - `jina-reranker-v1-tiny-en`：最快的重排序模型，适合可靠地对大量文档进行排序，8K 上下文，33M 参数
  - `jina-reranker-v1-turbo-en`：快速推理速度和准确相关性分数的最佳组合，8K 上下文，37.8M 参数
  - `jina-reranker-v1-base-en`：我们的第一个重排序模型，最大化搜索和 RAG 相关性，8K 上下文，137M 参数
  - `jina-colbert-v1-en`：英文 ColBERT 模型，8K 上下文，137M 参数
</ParamField>

<ParamField body="query" type="string" required>
  搜索查询文本，用于与候选文档进行比较
</ParamField>

<ParamField body="top_n" type="integer">
  要返回的最相关文档数量。默认返回所有文档
</ParamField>

<ParamField body="documents" type="array" required>
  候选文档数组，将根据与查询的相关性进行重新排序
</ParamField>

<ParamField body="return_documents" type="boolean" default="false">
  是否在响应中返回重新排序的文档内容。设置为 true 将返回文档文本，false 则只返回索引和分数
</ParamField>

成功的响应包含以下字段：
- `model`: 使用的模型名称
- `results`: 重排序结果数组，按相关性得分降序排列，每个元素包含：
  - `index`: 原始文档数组中的索引位置
  - `relevance_score`: 相关性分数，介于 0-1 之间，越高表示与查询越相关
  - (可选)`document`: 如果 `return_documents` 设为 true，则包含文档内容
- `usage`: 使用信息
  - `total_tokens`: 此请求处理的总 Token 数

## 三、深度搜索 (DeepSearch)
DeepSearch 结合了搜索、阅读和推理能力，直到找到最佳答案。它完全兼容 OpenAI 的 Chat API 格式，只需将 `api.openai.com` 替换为 `aihubmix.com` 即可开始使用。  
流式调用 (stream) 会返回思考过程。

### 请求参数

<ParamField body="model" type="string" required>
  模型名称，可用模型列表：
  - `jina-deepsearch-v1`：默认模型，搜索、阅读和推理直到找到最佳答案
</ParamField>

<ParamField body="stream" type="boolean" default="true">
  是否启用流式响应。强烈建议保持此选项开启，DeepSearch 请求可能需要较长时间完成，禁用流式可能导致 '524 超时' 错误
</ParamField>

<ParamField body="reasoning_effort" type="string" default="medium">
  限制推理模型的推理精力。当前支持的值有 `low`（低）、`medium`（中）和 `high`（高）。降低推理精力可以加快响应速度，减少响应中用于推理的 token，其中：
  - `low`：基本推理和搜索简单查询（最大 500K tokens/req）
  - `medium`：中等推理和搜索深度（最大 1M tokens/req）
  - `high`：最大推理和搜索复杂查询（最大 2M tokens/req）
</ParamField>

<ParamField body="budget_tokens" type="integer">
  确定 DeepSearch 处理过程允许使用的最大 Token 数。更大的预算可以通过对复杂查询进行更全面的搜索来提高响应质量。此参数会覆盖 `reasoning_effort` 参数
</ParamField>

<ParamField body="max_attempts" type="integer" default="1">
  DeepSearch 过程中解决问题的最大重试次数。较大的值允许 DeepSearch 通过使用不同的推理方法和策略重试解决问题。此参数会覆盖 `reasoning_effort` 参数
</ParamField>

<ParamField body="no_direct_answer" type="boolean" default="false">
  强制模型进行进一步思考/搜索步骤，即使查询看起来很简单。这在确定查询总是需要 DeepSearch 而不是琐碎问题（如"1+1=?"）的场景中很有用
</ParamField>

<ParamField body="max_returned_urls" type="integer">
  在最终答案/块中包含的 URL 的最大数量。URL 按相关性和其他重要因素排序
</ParamField>

<ParamField body="structured_output" type="boolean" default="false">
  启用结构化输出，确保模型最终答案与您提供的 JSON 模式匹配
</ParamField>

<ParamField body="good_domains" type="array">
  内容检索时被赋予更高优先级的域名列表。对于特定领域的高质量来源很有用
</ParamField>

<ParamField body="bad_domains" type="array">
  内容检索时严格排除的域名列表。通常用于过滤已知的垃圾站点、低质量或不相关的网站
</ParamField>

<ParamField body="only_domains" type="array">
  内容检索时专门包含的域名列表。所有其他域名将被忽略。适用于特定领域的搜索
</ParamField>

<ParamField body="messages" type="array" required>
  用户与助手之间的对话消息列表。支持多种类型（模态）的消息，如文本 (.txt, .pdf)、图像 (.png, .webp, .jpeg) 等。文件大小最大支持 10MB
</ParamField>

### 多模态消息格式
DeepSearch 支持多种类型的消息格式，可以包含纯文本（message）、文件（file）和图像（image）。以下是不同格式的示例：

#### 1. 纯文本消息
```json
{
  "role": "user",
  "content": "hi"
}
```

#### 2. 带有文件附件的消息
```json
{
  "role": "user",
  "content": [
    {
      "type": "text",
      "text": "what's in this file?"
    },
    {
      "type": "file",
      "data": "data:application/pdf;base64,JVBERi0xLjQKJfbk...", // PDF 文件的 base64 编码
      "mimeType": "application/pdf"
    }
  ]
}
```

#### 3. 带有图像的消息
```json
{
  "role": "user",
  "content": [
    {
      "type": "text",
      "text": "what's in the image?"
    },
    {
      "type": "image",
      "image": "data:image/webp;base64,UklGRoDOAAB...", // 图像的 base64 编码
      "mimeType": "image/webp"
    }
  ]
}
```

所有文件和图像必须以数据 URI 格式（data URI）提前编码，最大支持 10MB。

# Use stream=True in the request to handle streaming responses
response = requests.post(url, headers=headers, json=data, stream=True)

# Check if the request was successful
if response.status_code == 200:
    # Iterate over the response content line by line
    for line in response.iter_lines():
        if line:
            # Decode line and print (assuming SSE format: "data: {...}")
            decoded_line = line.decode('utf-8')
            if decoded_line.startswith('data: '):
                # Handle potential end signal like "data: [DONE]"
                if decoded_line[len('data: '):].strip() == '[DONE]':
                    print("\nStream finished.")
                    break
                try:
                    # Extract the JSON part
                    json_data = json.loads(decoded_line[len('data: '):])
                    # Process the JSON data
                    if 'choices' in json_data and len(json_data['choices']) > 0:
                        delta = json_data['choices'][0].get('delta', {})
                        content_to_print = delta.get('content') or delta.get('reasoning_content') # Check both fields

                        if content_to_print:
                            print(content_to_print, end='', flush=True)
                except json.JSONDecodeError:
                    # Ignore lines that are not valid JSON after "data: "
                    # print(f"\nCould not decode JSON from line: {decoded_line}")
                    pass # Optionally log or handle non-JSON data lines if needed
            # Handle lines that don't start with "data: " if necessary
            # else:
            #    print(f"Received non-data line: {decoded_line}")

    print() # Add a newline at the end
elif response.status_code == 401:
     print(f"Error: {response.status_code} - Unauthorized. Please check your API key.")
     try:
         print(response.json())
     except json.JSONDecodeError:
         print(response.text)
else:
    print(f"Error: {response.status_code}")
    try:
        print(response.json()) # Print error details if available
    except json.JSONDecodeError:
        print(response.text) # Print raw text if not JSON
