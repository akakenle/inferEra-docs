<Update label="2025年2月">
  ## 新しい設定スキーマ `docs.json`

  より良い多段階バージョン管理、より簡単な視覚的理解、より一貫した用語をサポートするために、`mint.json`の代替として新しい`docs.json`スキーマを導入しました。変更点の詳細については、[ブログ](https://mintlify.com/blog/refactoring-mint-json-into-docs-json)をご覧ください。



  `mint.json`から`docs.json`へのアップグレードは、以下の手順で行います。

  1. CLIが最新バージョンであることを確認します。

  ```
  npm i mintlify@latest -g
  ```

  2. ドキュメントリポジトリで以下を実行します。

  ```
  mintlify upgrade
  ```

  3. 古いmint.jsonファイルを削除し、変更をプッシュします。

  ## CIチェック

  壊れたリンクを見つけたり、スペルや文法の問題を検出したり、独自のVale設定で記述スタイルを強制したりするために、ドキュメントを自動的にリンティングします。詳細については、[ドキュメント](settings/ci)をご覧ください。

  ## LLMの.mdサポート

  すべてのドキュメントページが、プレーンなMarkdownファイルとして自動的に利用できるようになりました。URLに`.md`を追加するだけです。これにより、LLMがドキュメントから個々のページを簡単に取り込めるようになります。

  ## その他のテーマ

  ドキュメントの外観を変更するための新しい[組み込みテーマ](/themes)です。[docs.jsonファイル](/settings/global)を介して設定します。

  現在利用可能：

  * Maple

  * Palm

  * Willow

  ## その他の改善点
  - [テクニカルライティングガイド：](https://mintlify.com/guides/introduction)対象読者の調査、コンテンツの種類、ライティングのヒントなど、テクニカルドキュメント作成のベストプラクティス。
  - [ドロップダウンコンポーネント](/navigation/divisions#dropdowns)：タブやアンカーに加えて、ドロップダウンでナビゲーションを整理します。
  - [AI構文修正ツール](https://x.com/ricardonunez_io/status/1892334887644123192)：ウェブエディタは解析エラーを検出し、AIを使用して修正を提案します。
  

</Update>

  ## コード行のハイライト

  言語識別子の後に特別なコメントを追加することで、ドキュメント内のコード行をハイライトし、重要な部分を強調して注目を集めることができるようになりました。中括弧`{}`を使用し、行番号または範囲をコンマで区切って指定します。

  ```javascript Line Highlighting Example {1,3-5}
  const greeting = "Hello, World!";
  function sayHello() {
    console.log(greeting);
  }
  sayHello();
  ```

  ````md
  ```javascript Line Highlighting Example {1,3-5}
  const greeting = "Hello, World!";
  function sayHello() {
    console.log(greeting);
  }
  sayHello();
  ```
  ````

  コードブロックにライトモードのバリアントが追加されました。これは、`mint.json`に以下を追加することで有効にできます。
